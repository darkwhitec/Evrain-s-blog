## 部署方式

### 方式一：本地手动部署

```bash
npm run deploy
```

自动执行 `npm run build` 构建，然后用 cwRsync 增量上传到服务器。

> 脚本：`deploy.sh` — 使用 cwRsync 自带的 `rsync.exe` + `ssh.exe` 避免 Windows 兼容问题。

### 方式二：GitHub Actions 自动部署

推送代码到 `main` 分支后，GitHub Actions 自动构建并部署：

```
git add .
git commit -m "描述"
git push origin main
```

> 配置文件：`.github/workflows/deploy.yml`
> 推送后约 30 秒完成。可在 https://github.com/Ev-rain/My-blog/actions 查看状态。

### 方式三：GitHub 手动触发

在 https://github.com/Ev-rain/My-blog/actions/workflows/deploy.yml 点击 "Run workflow"。

## 注意事项

- ⚠️ **永远不要删除或修改** 服务器上的 `.user.ini` 文件，它是宝塔面板的保护文件
- rsync 已配置 `--exclude=.user.ini`，避免触及 `.user.ini`
- 系统自带 Nginx（`/etc/nginx/`）没有运行，实际运行的是宝塔自带的 Nginx（`/www/server/nginx/`）
- SSL 证书由宝塔面板管理
